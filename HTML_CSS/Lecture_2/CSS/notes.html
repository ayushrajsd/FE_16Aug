Combinators

In CSS, combinators are used to select HTML elements based on their relationship to other elements. Combinators allow you to create more specific and targeted CSS selectors by specifying the relationship between different elements in the HTML structure. There are four types of combinators in CSS:

Descendant Combinator

Descendant Combinator (space): Selects an element that is a descendant of another specified element. It matches any element that is inside the specified element, no matter how deeply nested. Example: div p { /* Selects

elements that are descendants of

elements */ }

Child(Immediate) combinators

Child Combinator (>): Selects an element that is a direct child of another specified element. It matches only if the second element is an immediate child of the first element. Example: ul > li { /* Selects li elements that are direct children of ul elements */ }

Adjacent Sibling Combinators

Adjacent Sibling Combinator (+): Selects an element that is the immediate next sibling of another specified element. It matches only if the second elemen` t comes immediately after the first element. Example: h2 + p { /* Selects p elements that come immediately after h2 elements */ }

General Sibling (~)

General Sibling Combinator (~): Selects elements that are siblings of another specified element and come after it. It matches any element that is a sibling of the first element, as long as it appears after the first element.

h3 ~ p { /* Selects p elements that are siblings of h3 elements and appear after them */ }
